# Security Policy / 安全策略

## Supported Versions / 支持的版本

| Version | Supported |
|---------|-----------|
| 4.x (Beta) | :white_check_mark: |
| < 4.0 | :x: |

## Reporting a Vulnerability / 报告漏洞

We take the security of Contract Comparator seriously. If you believe you have
found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

Instead, please report security vulnerabilities via email to:

**info@numboxhub.com**

You should receive a response within **48 hours**. If you do not receive a
response, please follow up to ensure we received your original message.

### Information to Include

Please provide the following information in your report:

- Type of issue (e.g., buffer overflow, SQL injection, XSS, etc.)
- Full paths of source file(s) related to the issue
- Location of affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step reproduction instructions
- Proof-of-concept or exploit code (if available)
- Impact of the issue, including how an attacker might exploit it

### What to Expect

1. We will acknowledge receipt within 48 hours.
2. We will investigate and provide an initial assessment within 5 business days.
3. We will keep you informed of progress toward a fix.
4. When the fix is released, we will credit you (if desired) in the changelog.

## Preferred Languages / 首选语言

We prefer all communications in **Chinese** or **English**.

## Policy / 政策

We follow the principle of [Responsible Disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure).

We request that you:
- Allow us reasonable time to fix the issue before public disclosure
- Make a good faith effort to avoid privacy violations, data destruction,
  and interruption of our services
- Do not exploit the vulnerability beyond what is necessary to demonstrate the issue

## Scope / 范围

This security policy applies to:

- The Contract Comparator application code
- Docker images and deployment configurations
- Official documentation and configuration templates

The following are **out of scope**:

- Third-party dependencies (report to their respective maintainers)
- Issues in user-deployed environments that do not originate from our code
- Theoretical attacks without a practical proof of concept

## PGP Key / PGP 密钥

If you need to encrypt your report, please request a PGP key via email.
